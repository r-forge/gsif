riodoce.grids {GSIF}R Documentation

Rio Doce study area grids at 1/120 arcdegrees (1 km) resolution

Description

A list of soil covariates prepared for the Rio Doce catchment in Brazil (71,492 grid nodes).

Usage

str(riodoce.grids)

Format

Data frame contains the following columns:

TAXBRC3

factor; dominant soil class from a 1:600,000 scale polgyon map, and based on the Brazilian Soil Classification System

GEOGLS3

factor; parent material class from the geological map

DEMSRT3

numeric; SRTM DEM

SLPSRT3

numeric; slope in degrees based on the SRTM DEM

TWISRT2

numeric; SAGA Topographic Wetness Index based on the SRTM DEM

INSSRT3

numeric; potential incoming solar radiation derived in SAGA GIS

EV1MOD3

numeric; first principal component of the monthly MODIS EVI time series data

EV2MOD3

numeric; second principal component of the monthly MODIS EVI time series data

TD1MOD3

numeric; first principal component of the 8-day MODIS day-time LST time series data

TN1MOD3

numeric; first principal component of the 8-day MODIS night-time LST time series data

x

numeric; x-coordinate in the UTM 23S coordiante system with GRS67 ellipsoid

y

numeric; y-coordinate in the UTM 23S coordiante system with GRS67 ellipsoid

Details

Soil classes ("AR", "CH", "CX", "CY", "LV", "LVA", "PV", "PVA", "RL"; Brazilian Soil Classification System) were obtained from the General soil map of Minas Gerais at scale 1:600,000. Land surface parameters: potential incoming solar radiation, SAGA Topographic Wetness Index and slope map were derived in SAGA GIS using the 1 km resolution SRTM DEM obtained from the Consultative Group on International Agricultural Research (CGIAR). Principal components for the Enhanced Vegetation Index (EVI, MOD13A3 product) and Land Surface Temperature (LST, MOD11A2 product) images were derived using the 8-day and monthly time-series MODIS images.

Note

Listed layer follow a standard naming convention used by WorlGrids.org (the standard 8.3 filename convention with at most eight characters): first three letter are used for the variable type e.g. DEM (digital elevation model); the next three letters represent the data source or collection method e.g. SRT (SRTM); the 6th character is the effective scale e.g. 3 indicates the 2nd standard scale i.e. 1/120 decimal degrees (about 1 km).

Examples

## Not run: # Load the data from GSIF page:
download.file("http://gsif.r-forge.r-project.org/riodoce.grids.rda", "riodoce.grids.rda")
load("riodoce.grids.rda")
gridded(riodoce.grids) <- ~x+y
utm.csy <- "+proj=utm +zone=23 +south +ellps=GRS67 +units=m +no_defs"
proj4string(riodoce.grids) <- CRS(utm.csy)
# plot maps:
col = grey(rev(seq(0.1,0.975,0.025)))
names(riodoce.grids)
spplot(riodoce.grids[9:10], col.regions=col)
# write to a GIS format:
writeGDAL(riodoce.grids["EV1MOD3"], "EV1MOD3.tif", "GTiff", mvFlag=-99999)

## End(Not run)
